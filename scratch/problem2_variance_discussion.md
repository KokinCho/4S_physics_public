# 周辺分散における Bagging と Random Forest の床値の差が小さい理由

周辺分散の測定において、アンサンブル数 $B=256$ に達した時点の予測値の分散 $V[\hat{f}(x_i)]$ は以下のようになっています。

*   **Bagging**: 約 0.869
*   **Random Forest**: 約 0.818

この差（約 0.05）は、一見すると理論的な期待（Random Forest が Bagging より大幅に分散を下げる）に比べて非常に小さいように感じられます。この現象は、以下の2つの統計的な要因によって整合的に説明できます。

---

### 1. 個々の木のバリアンス $\sigma^2$ と相関 $\rho$ のトレードオフ

アンサンブルモデル全体の予測値の分散は、以下の理論式で表されます。

$$
V[\hat{f}(x_i)] = \rho \sigma^2 + \frac{1 - \rho}{B} \sigma^2
$$

ここで、床値（$B \to \infty$ の極限）は第一項の $\rho \sigma^2$ によって決定されます。

*   **Bagging**:
    *   すべての特徴量を使って木を構築するため、個々の木の予測能力が高く、個々の木のバリアンス $\sigma^2_{\mathrm{Bag}}$ は小さくなります。
    *   しかし、構造が似通るため相関 $\rho_{\mathrm{Bag}}$ は大きくなります。
*   **Random Forest**:
    *   特徴量をランダムに 60% に制限するため、木同士の予測値の相関 $\rho_{\mathrm{RF}}$ は小さくなります（独立性が高まる）。
    *   しかし、特徴量を制限されることで個々の木が「質の悪い予測器」になり、個々の木のバリアンス $\sigma^2_{\mathrm{RF}}$ は**大きくなります**。

実際、実験結果の $B=1$（アンサンブル無しの単一決定木）における分散を比較すると、このトレードオフが数値に現れています。

*   $B=1$ における Bagging の分散: **14.29755**
*   $B=1$ における Random Forest の分散: **15.48408**

Random Forest の方が個々の木の分散 $\sigma^2$ が約 1.2 ほど大きくなっています。
したがって、全体の床値 $\rho \sigma^2$ を考えると、Random Forest は相関 $\rho$ を下げたものの、個々の木の分散 $\sigma^2$ 自体が増大してしまったため、その掛け算である床値 $\rho_{\mathrm{RF}} \sigma^2_{\mathrm{RF}}$ の低下幅が、Bagging の $\rho_{\mathrm{Bag}} \sigma^2_{\mathrm{Bag}}$ に比べて一部相殺され、結果として**「差がわずか 0.05 程度に縮まった」**と考えられます。

### 2. データサイズ $N=395$ の有限性による影響

今回の Student Performance データセットのサンプルサイズは $N = 395$ と比較的小規模です。
訓練データの 80%（約 316 点）を試行ごとにサンプリングしますが、プールが小さいため、異なる試行における訓練データセット $\mathcal{D}_1$ と $\mathcal{D}_2$ のデータの重複度は必然的に非常に高くなります。

データ自体が強く重複している以上、どれだけ特徴量選択をランダムに制限（Random Forest）したとしても、データを通じて木同士が持つ相関（周辺共分散）はどうしても一定以下には下がりません。
このため、モデルに依存しない本質的な共分散が支配的になり、両者の床値が似たような値に収束したと考えられます。
